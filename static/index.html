Masterprojekt - Realtime Web-Communication

WebSocket Evaluation

The prototypes aim to examine the capacity, performance and limits of the WebSocket protocol.

Prototype-One - Testcase 1

Arbitrary Data Serverpush - a simple ws-server: random text data is being pushed to the client.

Test-Setup with JMeter: Long-lived and mostly idle connections. The number of connections varies from 1,000 to 50,000, message size from 10 to 4096 bytes, and the frequency of messages from 0.1 to 10 seconds.

Prototype-One - Testcase 2

Arbitrary Data Serverpush - a simple ws-server: random text data is being pushed to the client.

Test-Setup with JMeter: Short-lived but highly active connections. The number of concurrent connections is constant at 500, with 50 messages per connection and no delay between messages. Message size ranged from 1 to 4096 bytes.

Prototype-Two

Push Notification Server - a simple socket.io-server: changed data is being pushed to the client.

Changed XML-file data causes Socket.IO-server to update this content.
